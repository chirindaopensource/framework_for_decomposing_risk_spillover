#
# Upper bound rationale:
#   <3.0.0 — a future 3.x release may alter the Array API or memory layout
#   assumptions that the stride-tricks code relies on.
#

# ==============================================================================
# SECTION 4 — ECONOMETRIC AND STATISTICAL MODELLING
# ==============================================================================
#
# statsmodels >= 0.14.0, < 1.0.0
# --------------------------------
# Provides:
#   • statsmodels.regression.quantile_regression  → QuantReg
#                                       Core QVAR building block: fits a single
#                                       quantile regression equation at tau
#                                       for one dependent variable.  The p-order
#                                       QVAR is constructed by iterating QuantReg
#                                       across all N equations.
#
#   • statsmodels.tsa.stattools        → adfuller
#                                       Augmented Dickey-Fuller unit-root test;
#                                       confirms I(0) stationarity of all 39
#                                       log-return series before QVAR estimation.
#
#   • statsmodels.tsa.vector_ar.var_model → VAR
#                                       Used for BIC-based global lag selection
#                                       (p = argmin BIC over candidate_lags=[1..5])
#                                       prior to rolling QVAR estimation.
#
# Lower bound rationale:
#   0.14.0 — the first release in the 0.14.x series that stabilised the
#   QuantReg solver (IRLS) after the statsmodels 0.13 regression-fitting
#   refactor; also the first version compatible with pandas 2.x.
#
# Upper bound rationale:
#   <1.0.0 — statsmodels has not yet reached a semantic-versioned 1.x release;
#   this cap prevents absorption of any potential breaking API restructuring
#   if/when a 1.0.0 milestone is published.
#
# Tested: 0.14.6
#
statsmodels>=0.14.0,<1.0.0

# ==============================================================================
# SECTION 5 — NETWORK TOPOLOGY AND GRAPH THEORY
# ==============================================================================
#
# networkx >= 3.1, < 4.0.0
# -------------------------
# Provides:
#   • networkx                        — directed weighted graph construction
#                                       (DiGraph) for the dense spillover network
#                                       and the sparsified disparity-filter backbone
#
# Specific API symbols used:
#   nx.DiGraph                        — directed graph representation of the
#                                       39-node connectedness network
#   nx.algorithms.isomorphism.DiGraphMatcher
#                                       exact VF2 subgraph isomorphism for
#                                       directed triadic motif enumeration
#                                       across all 13 non-isomorphic 3-node
#                                       subgraph configurations (Milo et al. 2002)
#   nx.algorithms.community            — modularity / community detection used
#                                       for sector-block validation of the backbone
#
# Lower bound rationale:
#   3.1 — NetworkX 3.x rewrote the VF2 isomorphism engine (vf2userfcndict)
#   for correctness and performance; the 2.x isomorphism code had a known
#   edge-case bug with directed multigraph induced subgraph matching.
#   The 3.1 release also formalised the DiGraphMatcher.subgraph_isomorphisms_iter
#   API that the motif enumeration loop depends on.
#
# Upper bound rationale:
#   <4.0.0 — a future 4.x release may alter the isomorphism iterator API
#   return type, which the motif-counting G-tries implementation unpacks
#   as a dict mapping.
#
# Tested: 3.6.1
#
networkx>=3.1,<4.0.0

# ==============================================================================

#                                       degree-preserving null model generation
#                                       (1 000 randomised networks × directed

#   count distributions without holding all 1 000 results in memory at once.
#
# Upper bound rationale:
#   <2.0.0 — joblib 2.x has not yet been released; the cap prevents absorption
#   of any breaking changes to the Parallel/delayed call signature.
#
# Tested: 1.5.3
#
joblib>=1.3.0,<2.0.0
